Steps for Getting Started

Install Trunx.IO CLI Wallet
JS binary/module

After installing Trunx.IO CLI wallet binary, you will be able to use the trunxcli executable directly on your local machine to interact with the backend wallet database. Developers also have the ability to import the Trunx.IO CLI wallet module in their own apps/scripts as well as create their own plugins for Trunx.IO CLI wallet.

  • Highly Documented
  • Tech Support Available
  • Very User Friendly
  • Fully Open Source

Start the Backend
Database Service

The backend database service is the brains of the Trunx.IO CLI Wallet. It is a long-running process responsible for the database management, administration, and security. For developers, Trunx.IO CLI wallet can be imported as a module to communicate directly with the backend database service from other apps/scripts.

  • Run Attached for Debugging
  • Managed Databases Coming Soon
  • Reliable and Robust
  • Fully Open Source

Create Your First
Trunx.IO CLI Wallet

With the backend service running, now it is time to create your wallet and get ready to have the power of EOSIO at your fingertips anywhere you go. Simply call the `db:unlock` command with the password you wish to encrypt your new database with. Be sure to always double check your passwords, backup your keys/databases, and test your login information before storing sensitive information inside.

  • AES-256-GCM Encryption
  • Configurable Auto-timeout Lock
  • Easily Manage Multiple Databases
  • Backend Database Audit Log

Import Your EOSIO
Keys and Accounts

Once your new Trunx.IO CLI wallet database is created, finish up by adding your EOSIO chains, keys, and accounts. Alternatively, you may choose to import an existing wallet backup from any of the following wallets: Trunx.IO, Anchor, Scatter. Enjoy your newly gained freedom amongst the EOSIO ecosystem (more chain integrations coming soon!).

  • Derive Keys from Mnemonic
  • Import Backups from Scatter/Anchor
  • Manually Import Keys
  • Fetch Accounts via Key

feature

Detailed Help
Documentation

Quality, in-depth documentation is available for each and every command present in the Trunx.IO CLI Wallet. Simply add --help to any command to see this incredibly useful output.

Having highly detailed documentation readily available helps enable our users to have a more fluid experience, and helps keep our devs away from tech support so they have more time to build awesome new features.

feature

Cross-Platform
and Fully Open Source

Trunx.IO CLI Wallet is coded in JavaScript on top of the OCLIF framework making it modern, reliable, and most importantly, Trunx.IO CLI Wallet works on ANY operating system regardless the chip architecture.

Oh and be sure to swing by the Trunx.IO GitHub repository because all of the Trunx.IO codebase is fully open source. If you have any questions or concerns, or even suggestions on improving stability, security, and reliability of Trunx.IO please do not hesitate to create an issue report on the GitHub issue tracker.

feature

Manage Multiple
EOSIO Wallets

With the ability to export the wallet database to a backup file, import backups into the database, and easily manage multiple named wallet databases, Trunx.IO CLI Wallet really is the most well-rounded wallet solution for EOSIO.

Quickly migrate your data from other EOSIO based wallets such as Anchor and Scatter so you don't have to spend time fighting with several different applications that achieve the same goal.

In the future, we hope to provide the EOSIO community with many exciting and useful plugins for any EOSIO blockchain network. Plugins for specific contracts, workflows, etc. There is no limit to the functionality that you can add to Trunx.IO CLI Wallet except your own imagination.
